# 用户管理后台首页示例

本示例基于本地 `SaaS Design Spec Plugin` 的规则生成，目标是展示插件在真实页面需求下的输出效果。

## 输入需求

```md
按我的 SaaS design spec，设计一个用户管理后台首页，要有概览卡片、筛选、表格和批量操作。
```

## 页面定位

- 页面类型：搜索表格页 + 管理型工作台首页混合页
- 页面目标：帮助管理员快速总览用户状态，并高效完成筛选、查看、批量管理等操作
- 目标用户：运营管理员、系统管理员、客服主管
- 使用频率：高频
- 设备优先级：桌面优先

## 应用的设计规则

- 使用固定顶部导航和左侧菜单的企业级后台壳层
- 页面内容采用 `16px / 24px` 的节奏组织，信息紧凑但易扫读
- 核心操作颜色采用蓝色主行动线，次级操作保持克制
- 概览、筛选区、表格区全部用卡片化容器承载，减少视觉噪音
- 表格操作强调效率，批量操作和状态标签优先保证可辨识性

## 页面结构

### 1. 页面头部

- 左侧：
  - 面包屑：`首页 / 用户管理`
  - 页面标题：`用户管理`
  - 一行辅助说明：`统一管理平台用户、角色状态与账号行为`
- 右侧：
  - 主按钮：`新增用户`
  - 次按钮：`导出用户`
  - 轻量按钮：`更多操作`

### 2. 概览卡片区

采用 4 张横向概览卡片，放在页面主体最上方。

- 卡片 1：`用户总数`
  - 数值：`128,540`
  - 辅助信息：`较昨日 +1.8%`
- 卡片 2：`本月新增`
  - 数值：`2,846`
  - 辅助信息：`自然注册占 72%`
- 卡片 3：`异常账号`
  - 数值：`63`
  - 辅助信息：`需优先处理`
- 卡片 4：`待审核用户`
  - 数值：`214`
  - 辅助信息：`平均等待 3.2 小时`

设计要求：

- 卡片统一白底
- 标题使用较弱文本色，核心数字使用主文本色
- 涨跌、异常、提醒使用语义色，但不大面积铺色

### 3. 筛选区

概览卡片下方放一个完整筛选卡片。

建议筛选项：

- 用户 ID / 用户名 / 手机号搜索框
- 用户状态下拉：`全部 / 正常 / 禁用 / 待审核 / 异常`
- 用户角色下拉：`全部 / 超级管理员 / 运营 / 客服 / 访客`
- 注册时间范围
- 最近登录时间范围
- 所属组织 / 部门

右侧操作：

- 主操作：`查询`
- 次操作：`重置`
- 轻量操作：`展开更多筛选`

设计要求：

- 筛选表单放在表格上方
- 对齐整齐，行距克制
- 默认展示常用筛选项，更多条件通过展开补充

### 4. 表格工具栏

筛选区下方紧接表格工具栏。

左侧：

- 已选提示：`已选择 18 项`
- 批量操作按钮：
  - `批量启用`
  - `批量禁用`
  - `批量分配角色`
  - `批量导出`

右侧：

- 列设置
- 刷新
- 密度切换
- 视图保存

设计要求：

- 当未勾选任何行时，批量操作弱化或禁用
- 工具栏与表格属于同一块操作区，视觉上紧密关联

### 5. 主表格区

表格字段建议：

- 用户名
- 用户 ID
- 手机号 / 邮箱
- 角色
- 状态
- 所属组织
- 最近登录时间
- 注册时间
- 操作

状态标签建议：

- 正常：绿色语义标签
- 禁用：灰色标签
- 待审核：金橙色标签
- 异常：红色标签

行级操作建议：

- 查看详情
- 编辑
- 重置密码
- 启用 / 禁用
- 更多

设计要求：

- 表格行高适中，支持高密度扫读
- 标签颜色必须服务于状态识别，不能喧宾夺主
- 行级操作保持稳定顺序，避免每行都不一样

### 6. 右侧详情抽屉（可选联动）

点击用户名或“查看详情”后打开右侧抽屉，不强制跳新页。

抽屉内容：

- 基础信息
- 角色与权限
- 登录记录摘要
- 风险与异常标记
- 最近操作记录

设计要求：

- 首页完成“筛选 + 浏览 + 快速处理”
- 深一层的信息通过抽屉承接，提高效率

## 页面状态设计

### 默认态

- 显示概览卡片、筛选区、工具栏和表格数据

### 加载中

- 概览卡片显示骨架屏
- 表格显示行骨架
- 筛选区保持可见但按钮 loading

### 空状态

- 表格区域显示空状态插画/图标
- 文案：`当前筛选条件下暂无用户数据`
- 提供按钮：`重置筛选`

### 错误状态

- 表格区显示局部错误反馈
- 文案：`数据加载失败，请稍后重试`
- 按钮：`重新加载`

## 推荐视觉效果总结

这个页面最终应该呈现为：

- 一个标准企业级后台首页，而不是营销页
- 先给管理员“看全局”的安全感，再给“做操作”的高效率
- 视觉上克制、清晰、稳定
- 信息密度高，但通过卡片、筛选、表格分层处理后仍然轻松可扫读

```md
- 筛选区
- 表格工具栏
- 用户表格
- 批量操作

目标用户：
- 运营管理员
- 系统管理员

页面目标：
- 让管理员快速查看用户整体情况

要求：

- 最后总结本次设计应用了哪些规则
```
