# SaaS Design Review Checklist

Use this checklist after generation and during design review.

## 1. Page Family

- Does the page belong to the correct family
- Is the nearest example route chosen sensibly
- If hybrid, is the primary family still obvious

## 2. Shell And Hierarchy

- Is the shell stable and unobtrusive
- Is the page title clear
- Are top actions easy to find
- Is the main content hierarchy obvious at first glance

## 3. Language Consistency

- Is the page in the requested language mode
- Is there any mixed-language chrome that should not be mixed
- Is any default library copy still visible

## 4. Content Realism

- Do the KPIs fit the page goal
- Do the filters support a believable workflow
- Do the table columns support one coherent task
- Do the row and bulk actions make business sense

## 5. List Layout

- Are filters above the results
- Are filter rows wrapped intentionally
- Are utility actions separated from filter actions
- Are bulk actions separated from global CTA
- Would one more filter still fit cleanly

## 6. Component Consistency

- Are cards visually aligned
- Are buttons using the right priority
- Are tags semantically colored
- Are tables compact but readable

## 7. Spacing And Density

- Is section rhythm consistent
- Is card padding consistent
- Is the page dense without becoming cramped
- Are there any accidental gaps or crowded clusters

## 8. Accessibility And State Clarity

- Is text contrast acceptable
- Are focus or selected states identifiable
- Are clickable areas sufficiently clear
- Are loading, empty, and error states considered

## 9. Final Gate

If the answer to any of these is no, revise before presenting:

- Does it feel like one coherent product page
- Does it look production-plausible rather than like a component showcase
- Would an operator understand what to do first
